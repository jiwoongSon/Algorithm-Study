# 💡 Note
## 🗄️ 1. 자료구조 (Data Structure)
> *데이터를 효율적으로 저장하고 관리하는 방법*

### 🔹 해시 (Hash / Map)
- **개념:** Key-Value 구조로 데이터를 짝지어 저장하며, 데이터의 중복을 제거합니다.
- **특징:** 해시 함수를 통해 데이터에 접근하므로 검색, 삽입, 삭제의 시간 복잡도가 `O(1)`로 매우 빠릅니다.
- **활용:** "이름-전화번호" 매칭, 빈도수 계산, 특정 데이터 유무의 빠른 확인 (금융권 코딩테스트 빈출 유형)

### 🔹 스택 (Stack)
- **개념:** 나중에 들어간 데이터가 먼저 나오는 **LIFO(Last In First Out)** 구조입니다.
- **활용:** 괄호 짝 맞추기, 웹 브라우저 뒤로 가기, 실행 취소(Undo), 깊이 우선 탐색(DFS) 구현

### 🔹 큐 (Queue)
- **개념:** 먼저 들어간 데이터가 먼저 나오는 **FIFO(First In First Out)** 구조입니다.
- **활용:** 은행 창구 대기열, 프로세스 스케줄링, 너비 우선 탐색(BFS) 구현

### 🔹 우선순위 큐 (Priority Queue / Heap)
- **개념:** 들어온 순서와 상관없이 우선순위가 가장 높은(또는 낮은) 데이터부터 먼저 나옵니다.
- **활용:** 계속해서 변화하는 데이터 셋 안에서 최댓값이나 최솟값을 `O(log n)`의 속도로 빠르게 뽑아내야 할 때 사용합니다.

## 🔍 2. 탐색 및 정렬 (Search & Sort)
> *원하는 데이터를 정확하고 빠르게 찾아내거나 배치하는 방법*

### 🔹 완전 탐색 (Brute Force)
- **개념:** 컴퓨터의 빠른 연산 속도를 이용해 가능한 모든 경우의 수를 일일이 다 확인하는 직관적인 방법입니다.
- **활용:** 비밀번호 4자리 모두 입력해 보기, 순열 및 조합 생성, 예외 없이 100% 정답을 보장해야 할 때 사용합니다.

### 🔹 DFS & BFS (깊이 우선 탐색 & 너비 우선 탐색)
- **개념:** 그래프나 트리의 모든 노드를 한 번씩 방문하는 양대 알고리즘입니다.
- **활용:** - **DFS (재귀/스택):** 경로의 특징을 저장해야 하거나, 모든 조합을 탐색할 때 (예: 미로 탐색, 타깃 넘버)
  - **BFS (큐):** 최단 거리, 최단 시간을 구하거나 특정 영역의 크기를 구할 때 (예: 최단 경로 찾기, 네트워크 연결 상태)

### 🔹 이분 탐색 (Binary Search)
- **개념:** **정렬된 배열**에서 탐색 범위를 절반씩 줄여나가며 특정 값을 찾는 마법 같은 알고리즘입니다.
- **특징:** 시간 복잡도가 `O(log n)`으로, 1억 개의 데이터 중에서도 20여 번의 연산만으로 정답을 찾을 수 있습니다.

### 🔹 정렬 (Sorting)
- **개념:** 데이터를 특정 조건(오름차순, 내림차순, 커스텀 규칙)에 맞게 재배치합니다.
- **활용:** 자바의 `Arrays.sort()`, `Collections.sort()` 및 `Comparable`, `Comparator` 인터페이스를 활용한 다중 조건 정렬 숙지가 필수적입니다.

## 🚀 3. 심화 알고리즘 (Advanced)
> *Level 3 이상 도약을 위한 고효율 문제 해결 기법*

### 🔹 그리디 (Greedy / 탐욕법)
- **개념:** 미래를 내다보지 않고, 현재 상황에서 가장 좋아 보이는 최선의 선택을 반복하여 결과를 도출합니다.
- **활용:** 잔돈 최소 개수로 거슬러 주기, 회의실 배정 등 (단, 현재의 선택이 최종 결과에 악영향을 미치지 않는지 증명이 필요함)

### 🔹 동적 계획법 (DP / Dynamic Programming)
- **개념:** 크고 복잡한 문제를 작은 문제들로 나누어 풀고, 과거에 계산된 결과값(Memoization)을 저장해 두었다가 재활용하여 연산 속도를 기하급수적으로 줄입니다.
- **활용:** 피보나치수열, 배낭 문제 등 가장 추상적이고 난이도가 높은 파트입니다.

### 🔹 최단 경로 (Shortest Path)
- **개념:** 노드 간의 간선에 가중치(비용/시간)가 있을 때, 가장 적은 비용으로 도달하는 경로를 찾습니다.
- **활용:** 다익스트라(Dijkstra - 하나의 출발점), 플로이드-워셜(Floyd-Warshall - 모든 출발점) 알고리즘. 내비게이션 길 찾기 구현에 쓰입니다.

## 💼 4. 금융권 및 금융공기업 특화 필수 영역
> *수학적 알고리즘보다 '정확하고 예외 없는 로직 구현'에 초점을 맞춘 파트*

### 🔹 구현 / 시뮬레이션 (Implementation & Simulation)
- **특징:** 특별한 배경지식 없이도 풀 수 있지만, 문제에서 제시하는 길고 복잡한 "은행 업무 규칙"이나 "특정 시스템의 동작 로직"을 예외 없이 코드로 완벽하게 번역해야 합니다.
- **핵심:** 코드가 매우 길어지므로 변수 관리, 조건문 분기, 모듈화(함수 분리) 등 끈기와 집중력, 꼼꼼함이 합격을 좌우합니다.

### 🔹 문자열 처리 (String Processing)
- **특징:** 날짜/시간 계산, 고객의 로그 데이터 파싱, 특정 패턴의 텍스트 추출 및 치환 작업을 수행합니다.
- **핵심:** 문자열 슬라이싱, 자바의 `String` 메서드, `StringBuilder` 활용, 정규 표현식(Regex) 기초 지식이 필수적입니다.

### 🔹 SQL (데이터베이스 쿼리)
- **특징:** 데이터베이스에서 조건에 맞는 데이터를 추출하고 가공하는 능력으로, 금융권 IT 직무에서는 일반 알고리즘만큼이나 비중이 큽니다.
- **핵심:** 기본적인 CRUD 조작뿐만 아니라 `JOIN` (테이블 결합), `GROUP BY` & `HAVING` (데이터 그룹화 및 필터링), 서브 쿼리(Subquery) 작성 능력은 완벽하게 체화되어 있어야 합니다.
